# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repository Is

`SanskritRussian` is a static data + site repo: a full Sanskrit→Russian
translation glossary at three granularities (surface form, lemma/stem, verb
root), built from the 1,091,528-token word-aligned corpus in
[`SanskritLexicography/RussianTranslation/src/corpus_lexicon.jsonl`](https://github.com/gasyoun/SanskritLexicography/blob/master/RussianTranslation/src/corpus_lexicon.jsonl).
Published as data files + a searchable static front-end
(`index.html`) on GitHub Pages at <https://gasyoun.github.io/SanskritRussian/>.
**This repo has no build scripts of its own** — the generator pipeline lives
in the sibling `SanskritLexicography` repo; this repo is the published output
plus the browsing UI.

## Common commands

There is nothing to build here. To regenerate the data, run the pipeline in
the sibling repo (from `SanskritLexicography/RussianTranslation/src/`):

```sh
python build_dcs_maps.py            # DCS form->lemma + lemma->root maps (needs VisualDCS dcs_full.sqlite)
python build_surface_glossary.py    # Layer 1 (surface forms)
python build_rollup_glossaries.py   # pass 1: emits surface_dcs_misses.tsv (Vidyut input)
python build_vidyut_fallback.py <kosha_dir>   # lemmatize the DCS misses (needs vidyut.download_data())
python build_rollup_glossaries.py   # pass 2: folds in Vidyut + marker tier -> Layers 2 & 3
```

Two-pass bootstrap is required — the first rollup pass emits the DCS-miss
list the Vidyut stage consumes, so it must run once before and once after
the Vidyut fallback. Output files then get copied/published into this repo.

## Key directories / files

| Path | Purpose |
|---|---|
| `surface_glossary.tsv` + `surface/<X>.jsonl(.gz)` | **Layer 1** — 190,838 forms → Ru, with counts/registers/works/kinds. Split by initial SLP1 letter (26 parts) because the full `surface_glossary.jsonl` exceeds GitHub's 100 MB limit; `cat surface/*.jsonl` reconstructs it |
| `lemma_glossary.jsonl`/`.tsv` | **Layer 2** — 40,370 lemmas → Ru |
| `root_glossary.jsonl`/`.tsv` | **Layer 3** — 2,021 verb roots → Ru (verbal only — nominal stems stop at Layer 2 by design) |
| `md/{surface,lemma,root}/` | Human-readable per-initial-letter Markdown mirrors of each layer |
| `dcs_form2lemma.tsv`, `dcs_lemma2root.tsv` | DCS lemmatization maps consumed by the pipeline |
| `vidyut_form2lemma.tsv` | Vidyut fallback lemmatization for DCS-missed forms |
| `surface_dcs_misses.tsv` | Deterministic snapshot of DCS-unresolved forms — the stable input to the Vidyut pass |
| `surface_unresolved.tsv` | Forms no tier (DCS+Vidyut+marker) resolved — the failure-typology input |
| `ambiguity_homographs.tsv` | Forms whose top DCS lemmas span different POS or are near-equal in count |
| `index.html` | The searchable front-end over the root/lemma TSVs |
| `INTERLINKS.md` | Upstream/downstream repo relationships |
| `USE_CASES.md` | Intended query scenarios |

## Conventions

- **Filesystem-forced case folding**: bucket filenames under `surface/` are
  case-folded to upper (`a` and `A` share one file) because SLP1 is
  case-significant but Windows' filesystem is case-insensitive — separate
  `a`/`A` files would collide. The phonemic distinction is preserved inside
  each record's `slp1` field, not in the filename.
- **Scope is everything, n ≥ 1** — hapax legomena are kept, and both
  `kind=translation` and `kind=commentary` entries are kept (each record
  records which). Don't filter these out when consuming the data.
- **Homograph attribution**: a form's Russian distribution is attributed to
  its highest-count DCS lemma; alternates are logged (not double-counted) in
  `ambiguity_homographs.tsv`.
- **Every rollup record carries a `source` field** (`dcs`/`vidyut`/`marker`)
  for provenance — don't strip it when re-deriving data from these files.
- **Join-key normalization**: leading avagraha/apostrophe is stripped on both
  sides before joining (`'gacchat` == `gacchat`); a few corpus forms keep an
  internal space (e.g. `vācas pati`) and key on the spaced string.
- This repo is downstream-only for the data — do not hand-edit the `.tsv`/
  `.jsonl` glossary files here; fix the source pipeline in
  `SanskritLexicography/RussianTranslation/src/` and re-publish.

## What not to touch

- All glossary `.tsv`/`.jsonl`/`.jsonl.gz` files — generated by the pipeline
  in the sibling `SanskritLexicography` repo; hand-edits here will be
  overwritten and won't propagate upstream.
